# Vol-Fitter Technical Notes — Roadmap

A series of standalone LaTeX technical papers documenting the mathematics,
algorithms, assumptions, heuristics, hyperparameters (surfaced **and** hidden),
numerical optimizations and performance of the Vol-Fitter application.

## Decisions of record (2026-06-28, confirmed with the user)

1. **Rewrite comprehensive.** Each topic that already had a theory-pure `.tex`
   note (LQD, Local-Vol, Multi-Core SIV, graph, weights) is *superseded* by a
   fuller note that fuses theory + implementation + performance. The legacy
   `.tex` files in `Docs/` are kept as historical sources but are no longer the
   reference; the `Docs/notes/NN_*.tex` files are.
2. **~15 focused topic notes**, one standalone document per topic.
3. **Clean-math body + appendices.** The body reads as rigorous mathematics at
   an advanced-undergraduate level. Every note ends with **Appendix A — a
   hyperparameter atlas** (each knob, surfaced or hidden, with its default and
   role) and **Appendix B — performance** (the numerical optimizations and
   measured timings). No inline `file:line` references in the body.
4. **Fresh figures/tables from the code.** Numerical examples, plots and timing
   tables are generated by running the actual fitter / benchmarks
   (`backend/.venv`), not reused. Generators live in `Docs/notes/figures/` and
   are re-runnable.

## Conventions

- Shared preamble: `Docs/notes/volfit_preamble.sty` (macros, theorem envs,
  `tcolorbox` asides: `heuristic` / `perfbox` / `caution` / `example`).
- Figures: `Docs/notes/figures/gen_<topic>.py` → `Docs/notes/figures/*.pdf`,
  each script importable and deterministic (fixed seeds, no network).
- Audience: advanced maths undergraduate; rigorous but an entertaining read;
  "leave nothing behind"; emphasize what is genuinely original.
- Build: `latexmk -pdf NN_topic.tex` (MiKTeX present).
